## 任务
用户提供了新的 `QoS模板.docx`，要求：
1. 只使用这一个模板，用户不用选择模板
2. 用户需要选择终审级别

## 模板分析
新模板 `QoS模板.docx` 使用 `[[占位符]]` 标记，包含5个表格：
- Table 1: 设表08-1/2 复核卡片
- Table 2: 设表08-2/2 复核卡片续表
- Table 3: 设表09-1 所室级（√所（室））
- Table 4: 设表09-1 院级（√院级）
- Table 5: 设表09-1 公司级（√公司级）

占位符列表（9个）：
- `[[这里填写项目名称及阶段]]`
- `[[这里填写设计范围]]`
- `[[这里填写完整图名或文件名]]`
- `[[这里填写张数]]`
- `[[这里填写复核意见]]`
- `[[这里填写所所审意见]]`
- `[[这里填写院审审查意见]]`
- `[[这里填写公司审审查意见]]`
- `[[这里填写终审级别]]`

## 核心技术问题
模板中 `[[`、占位符文字、`]]` 被拆分到3个独立的 `<w:t>` 标签中：
```xml
<w:t>[[</w:t></w:r><w:r>...<w:t>占位符文字</w:t></w:r><w:r>...<w:t>]]</w:t>
```

### 调试过程
1. 首次尝试：`doc_xml.replace('[[占位符]]', value)` — 失败，因为占位符跨标签
2. 第二次尝试：正则 `<w:t[^>]*>\[\[</w:t>.*?<w:t[^>]*>\]\]></w:r>` — 失败，`]]` 后面不是 `</w:r>`
3. 第三次尝试：正则 `\[\[.*?\]\]` + 函数提取 `<w:t>` 文本 — 失败，因为 `[[` 在 block 开头，前面的 `<w:t>` 开标签被正则消费了，`re.findall` 只提取到中间的占位符文字，不含 `[[` 和 `]]`
4. **最终方案**：正则 `\[\[.*?\]\]` + 直接用提取到的文本作为占位符名（因为 `[[` 和 `]]` 之间的 `<w:t>` 文本就是纯占位符文字）
