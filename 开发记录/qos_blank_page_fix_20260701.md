# QoS 生成器空白页修复

## 目标
修复 QoS 文件生成器中"终审级别选院级或所级时，Word 中多出空白页"的问题。

## 根因分析
模板有 5 张 `<w:tbl>` 表格，表间结构：
```
[表格] → [注段(无分页符)] → [空分页段(含<w:br w:type="page"/>)] → [页脚段] → [下张表]
```
旧代码删除表格时，只在"表后间隔"中找分页符段，遇到有文字的段（如"注："段）就 break，没继续找后面的空分页段。导致：
- 被删表的前面的分页符段残留 → 强制分页 → 空白页
- 被删表的后面的分页符段也可能残留

## 修复方案
`remove_tables_by_level` 函数重写：
1. **删表前面**：从前一张表 end 到本表 start 的 gap 中，扫描所有 `<w:p>` 段落，找含 `w:type="page"` 的段（跳过"注："段等不含分页符的段），从该分页段开始删除
2. **删表后面**：同样扫描表后到下张表的 gap，找含分页符的段，一起删除
3. **从后往前**拼接，避免位置偏移

关键修改：遇到不含分页符的段落时 `continue` 继续找，而不是 `break` 停止。

## 测试结果
| 级别 | 表格数 | 分页符数 | 项目名 | 设计范围 |
|------|--------|---------|--------|----------|
| 所级 | 3 ✅ | 1 ✅ | ✅ | ✅ |
| 院级 | 4 ✅ | 2 ✅ | ✅ | ✅ |
| 公司级 | 5 ✅ | 3 ✅ | ✅ | ✅ |

- 所级：1个分页符 → 2页
- 院级：2个分页符 → 3页
- 公司级：3个分页符 → 4页
